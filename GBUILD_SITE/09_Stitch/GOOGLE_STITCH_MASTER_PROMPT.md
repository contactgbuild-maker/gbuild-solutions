# Google Stitch Master Prompt - GBuild Solutions

Design a premium, responsive company website for `gbuildsolutions.online`.

## Company

GBuild Solutions is a software, AI systems, web development, SaaS, and business digitalization company. It serves SMEs, startups, institutions, and businesses that need clear technical execution and useful digital systems.

## Creative Goal

Create a website that feels:

- premium
- interactive
- modern
- professional
- trustworthy
- technically advanced
- human-made

It must not feel AI-generated or like a generic agency template.

## Brand Rules

- no emojis
- no AI cliche visuals
- no robot heads, glowing brains, or cyberpunk interfaces
- no fake stats or fake client logos
- no decorative clutter
- no generic repeated SaaS section patterns
- design must feel intentional and human-made

## Recommended Visual Direction

Use a hybrid dark and light system:

- dark premium hero and selected high-impact sections
- calm light content sections for clarity and readability
- one dominant accent color based on deep teal
- subtle support tones only

## Color Direction

Suggested palette:

- graphite text and dark surfaces
- cloud-white or warm-neutral light base
- stone borders
- slate secondary text
- deep teal accent
- restrained sand detail tone

Use elegant gradients only where they improve depth and hierarchy. Avoid unreadable or noisy gradients.

## Typography

Use a premium sans-serif such as `Geist` or `Satoshi`.

Typography should feel:

- controlled
- professional
- modern
- sharp but readable

Do not use decorative futuristic fonts. Do not use cheap editorial cliches.

## Interaction Direction

- premium scrolling experience
- subtle motion, strong polish
- clean hover states
- tactile CTA interactions
- refined page transitions only if they remain fast

## 3D Direction

Use 3D only where it adds meaning. Possible directions:

- abstract system architecture object
- modular software blocks
- AI workflow or node structure
- dashboard/data cube

3D must look useful and premium, not decorative noise.

## Pages To Design

- homepage
- services page
- AI solutions page
- case studies page
- process page
- contact page
- mobile versions for all of them

## Content Structure Guidance

### Homepage

- strong hero with clear positioning
- short support copy
- primary and secondary CTA
- services overview
- why clients trust us
- systems/capability section
- GB Vertical ecosystem proof block
- case-study or proof section
- process snapshot
- final CTA

### Services Page

- clear service architecture
- concise service blocks
- business-readable outcomes
- CTA

### AI Solutions Page

- practical AI positioning
- workflow use cases
- business-focused diagrams
- CTA

### Case Studies Page

- real-feeling proof layouts
- challenge / solution framing
- room for screenshots and structured placeholders
- CTA

### Process Page

- Discover
- Define
- Design
- Build
- Launch
- Improve

### Contact Page

- professional intro
- low-friction form
- project-type selection
- short project summary field
- strong CTA

## Final Quality Target

The website should feel like a high-end software and AI company with real implementation depth. It should communicate engineering quality, business clarity, and modern execution without looking like a template or an AI-generated moodboard.
