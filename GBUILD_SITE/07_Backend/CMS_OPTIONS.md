# GBuild Solutions - CMS Options

## V1 Recommendation

No CMS required at launch.

Use file-based content for:

- services
- homepage sections
- about content
- case-study placeholders
- early blog if needed

## V2 Options

- Sanity
- Payload
- Supabase-backed custom admin

## Decision Logic

Choose a CMS only when:

- the founder needs regular non-code publishing
- blog volume becomes real
- case studies need easy editing
- admin ownership is clearly defined
