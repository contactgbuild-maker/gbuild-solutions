# GBuild Solutions - Database Plan

## V1 Database Need

Optional but recommended if lead handling should be structured from day one.

## Minimal V1 Data

- contact inquiries
- service interest
- timestamps
- lead source fields if analytics tagging is added

## Recommended Database Direction

- PostgreSQL
- Supabase or Neon

## Why

- scalable enough for later blog, case studies, and admin flows
- simple enough for early-stage site needs

## V2 Data Expansion

- blog posts
- case studies
- newsletter subscribers
- admin users
- lead status and notes
