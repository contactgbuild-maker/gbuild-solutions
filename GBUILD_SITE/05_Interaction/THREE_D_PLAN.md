# GBuild Solutions - 3D Plan

## 3D Role

3D should represent systems, architecture, and digital structure. It should never be used as random spectacle.

## Good Uses

- abstract modular blocks representing software systems
- data cube or layered dashboard form
- node network showing connected workflows
- architecture object in the hero or capability section

## Where To Use It

- homepage hero
- AI Solutions feature visual
- process or systems capability chapter

## Where Not To Use It

- every page hero
- service cards
- contact page
- dense mobile flows
- any place where performance or readability would suffer

## Style Rules

- matte, refined, premium surfaces
- restrained palette tied to the brand accent system
- no floating robot heads
- no sci-fi hologram overload

## Performance Rules

- use static or lightly animated fallback on mobile/low-power contexts
- cap scene complexity
- lazy-load 3D only where it adds real value
