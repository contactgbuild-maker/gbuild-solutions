# GBuild Solutions - Frontend Plan

**Date:** 2026-06-26

## 1. Frontend objective

Create a website that feels:

- premium
- calm
- structured
- modern
- human-designed

It must not look like a generic AI agency or boilerplate SaaS theme.

## 2. Page structure

Core page set:

- Home
- About
- Services
- AI Solutions
- Web & SaaS Development
- Business Digitalization
- Portfolio / Case Studies
- Process
- Blog / Insights
- Contact

## 3. Component system

### Core layout components

- site header
- site footer
- page hero
- section shell
- split content section
- CTA band

### Content components

- service card
- service detail block
- case-study card
- case-study highlight
- process step block
- FAQ item
- testimonial block

### Conversion components

- primary CTA button
- secondary CTA button
- contact form
- newsletter form
- sticky contact CTA on mobile if desired

## 4. Navigation plan

### Desktop

- left: logo
- center/right: main nav
- far right: primary CTA button

Recommended top nav:

- About
- Services
- Process
- Portfolio
- Insights
- Contact

### Mobile

- compact header
- controlled slide-over or full-screen menu
- visible contact CTA

## 5. Responsive behavior

- mobile-first layout logic
- all multi-column sections collapse cleanly to single column below tablet
- hero text must remain readable and not wrap awkwardly
- cards must stack with clear spacing
- forms must remain easy to complete on mobile

## 6. Design system guidelines

- light background base with occasional dark sections
- one primary accent color
- strong spacing scale
- disciplined typography
- restrained border and card usage
- section rhythm with clear breathing room

## 7. CTA components

Primary CTA use cases:

- Book a consultation
- Discuss your project
- Start with discovery

Secondary CTA use cases:

- Explore services
- View process
- See selected work

Rules:

- do not flood each section with multiple equal CTAs
- every page should have a clear primary action

## 8. Service cards

Each service card should contain:

- title
- concise business-oriented summary
- 2-4 capability hints
- small directional link

Avoid:

- giant icon-led tiles
- equal template cards repeated everywhere

## 9. Case-study cards

Each card should include:

- project title
- category or sector
- problem summary
- what was built
- optional outcome or proof note

If screenshots are unavailable, use:

- structured text-first case-study layouts

## 10. Process timeline

Recommended steps:

- Discover
- Define
- Design
- Build
- Launch
- Improve

UI behavior:

- strong vertical timeline on mobile
- horizontal or staggered timeline on desktop

## 11. Contact forms

Suggested fields:

- name
- company
- email
- service interest
- short project description
- optional timeline

Form principles:

- short
- credible
- low-friction
- easy to scan

## 12. Animations

Motion should be subtle and purposeful.

Recommended:

- section fade-and-rise on entry
- small stagger reveals
- button press feedback
- hover refinement on cards

Avoid:

- flashy hero motion
- endless floating AI shapes
- decorative animations with no communication value

## 13. Accessibility

- proper semantic headings
- high contrast
- focus-visible states
- keyboard accessibility
- touch targets minimum 44px
- no meaning carried by color alone
- reduced-motion support

## 14. Content density

Aim for:

- clear section hierarchy
- generous whitespace
- concise copy blocks
- fewer but stronger sections

Avoid:

- wall-of-text service pages
- 12-card feature grids
- overly compressed mobile layouts

## 15. Visual assets needed

- approved logo
- favicon
- selected brand palette
- optional UI screenshots
- optional ecosystem visual
- case-study imagery where approved

## 16. Recommended V1 frontend scope

Must-have:

- polished marketing pages
- service detail pages
- contact form
- basic portfolio structure

Can wait:

- advanced filtering
- animation-heavy visuals
- account/auth flows
- dashboard surfaces
